# Manga Read Flow

Manga Read Flow 是一个本地漫画翻译与基础嵌字工作流，帮助普通读者把自己合法持有的日文漫画图片转换为基础中文可读版本。当前仓库包含架构验证代码、单页视觉闭环的实现基础，以及可复用的实验工具；它还不是完整可交付产品。

项目不提供漫画搜索、抓取、下载、分发或发布，也不以专业汉化、复杂拟声词、艺术字和精修排版为目标。

## 当前入口

- 运行全部测试：`pytest -q`
- 运行产品集成测试：`pytest -q tests/integration`
- 产品代码：`src/manga_read_flow/`
- 可复用开发与实验工具：`tools/dev/`、`tools/experiments/`

仓库当前没有正式 Web 启动入口；最小单页 Web 预览属于 M1 待完成范围，不能把测试或实验脚本描述为产品入口。

## 目录职责

| 路径 | 职责 |
| --- | --- |
| `src/` | 产品代码与稳定领域边界 |
| `tests/` | 产品、集成和保留工具的自动验证 |
| `tools/` | 开发辅助与可复用实验 runner/evaluator |
| `docs/` | 编号化的当前事实来源 |
| `data/` | 本地数据策略；`data/local/**` 含真实素材与运行结果，不进入 Git |

## 编号体系

`100–170` 表示线性处理阶段，`200–240` 表示跨阶段控制，`300–360` 表示平台能力。文档文件名的后两位表示文档职责：`00` 当前设计、`10` 已知问题、`20` 实验索引、`30` 合同、`40` 阶段检查、`50` 决策记录、`90` 目录说明。编号只用于信息组织，不改变运行时状态机。
