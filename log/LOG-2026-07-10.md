### 2.2 Experiment design

实验被拆为：

#### Experiment A — Pure OCR

```text
人工 ground-truth container bbox
→ crop
→ manga-ocr
→ expected / normalized comparison
```

目的：

* 排除 Detection 误差；
* 单独评估 manga-ocr；
* 区分 OCR 错误和上游裁剪错误。

#### Detection-only

```text
完整 Page
→ PaddleOCR detection
→ prediction fragments
→ 与 semantic GT container 做几何归属评估
```

核心指标：

* GT region hit / miss
* region recall
* 每个 GT container 内的 prediction 数
* unmatched predictions
* fragmentation
* auxiliary text

普通 bbox IoU 没有被当成唯一主指标，因为 GT 是 semantic container，而预测通常是 line-level fragment。

#### Experiment B1 — Oracle container group

```text
PaddleOCR fragments
→ 使用 GT container assignment 分组
→ union crop
→ manga-ocr
```

该实验只用于估计“已知正确分组时”的上限，不能证明产品具备自动 grouping。

#### Experiment B2 — Native fragments

```text
每个 fragment 独立 crop
→ manga-ocr
→ 使用已知方向排序
→ 拼接
```

该实验用于判断 fragment OCR 和排序的可用性，但仍没有验证自动 semantic grouping。

### 5.4 P0 visual-context decision

本 Session 识别出漫画翻译的视觉语境风险：

* 日语经常省略主语、宾语和对话对象；
* 单纯 OCR 文本可能无法确定说话人、语气、动作和人物关系；
* 当前 SRS/HLD 没有正式 speaker_id、bubble-character association 或视觉翻译输入。

经过风险、硬件和范围评估，最终决定：

```text
P0 不引入 VLM。
```

P0 Page Translation 输入保持：

```text
长期 System Prompt
+ 当前页 OCR TextBlocks
+ reading_order / grouping
+ Project glossary
+ 有界前序已接受译文
→ 强文本 LLM
→ Page 级结构化 Translation JSON
```

理由：

* 通用 VLM 对漫画气泡、人物归属、漫画符号和复杂分镜理解不稳定；
* 8GB GPU 下本地 VLM 与文本模型并行代价过高；
* 视觉调用增加延迟、复杂度和错误传播路径；
* `YOLO + 最近人物` 只能形成弱提示，不能提供权威 speaker；
* 尚未有真实数据证明视觉缺失在大比例页面造成实质性误译。

明确不进入 P0：

```text
当前页图片作为 Translation 必需输入
VLM visual-context stage
正式 speaker_id
气泡—人物权威关联
角色检测与跨页跟踪
视觉情绪识别
```

视觉能力后置为条件性：

```text
Translation Context Necessity Spike
```

只有真实页面证明视觉缺失比例和影响足够大时，才重新评估按需 VLM、漫画专用模型或人工 review。

### 5.5 Text-only ambiguity policy

长期 Prompt 加入以下约束：

* 使用整页文本、阅读顺序、grouping、glossary 和历史译文做有限推断；
* 无法确认时保留原文中的主语和指代省略；
* 不得无依据补充姓名、性别、说话人、对象和人物关系；
* 只有歧义会实质改变译义时才输出 uncertainty。

允许的 uncertainty flags：

```text
context_ambiguous
pronoun_resolution_uncertain
speaker_context_uncertain
addressee_context_uncertain
ocr_uncertain
```

这些只是 Provider candidate evidence。

Provider 不得输出或决定：

```text
retry
warning
pause
block
accepted
```

正式 workflow decision 仍属于 QualityCheckService 和 WorkflowLoopEngine。

2. D 组两个 empty responses 发生在 `context-dependent` 和 `long-page`；
3. 对这些 fixture，C 与 D 的 payload 和 request hash 相同；
4. 因此失败不能归因于 previous context；
5. 无输出 block 应分类为 `NO_OUTPUT / NOT_EVALUABLE`，不应计为 translation `UNUSABLE`；
6. API response rate、schema-valid among responses 和 end-to-end valid rate 应分开统计；
7. `先輩 → 学长/学姐`、`渡す → 给他` 等无依据补全应计入 `unsupported_disambiguation`，而不是 0。

基于以上问题，主 Spike 的正确阶段状态被调整为：

```text
FURTHER_SPIKE
```

需要一个窄范围 Full Text Context stability follow-up，而不是重做整个 Page Translation Spike。

```text
Page Translation core: CONDITIONAL_GO
```

Validated:

* uncertainty flags remain weak evidence and cannot directly determine blocking;

Keep only the seam and provenance fields needed for future enablement.

```text
DISABLE_FOR_MVP
```

Meaning:

* P0 does not construct or send non-empty previous context by default;
* no schema or interface capability needs to be deleted;
* previous context can later become optional or compact;
* no third general-purpose previous-context Spike is justified now.
