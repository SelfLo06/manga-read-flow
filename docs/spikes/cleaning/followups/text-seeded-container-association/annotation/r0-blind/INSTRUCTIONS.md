# R0 双人盲标填写说明

## 1. 你要判断什么

对每张匿名 ROI，只根据图像本身判断：

1. 是否存在真实文字；
2. 图中有几个完整或可辨识的文字组；
3. 哪些文字组属于同一个容器；
4. 容器的可见边界在哪里；
5. 边界缺损、遮挡或两个容器接触时，是否需要虚拟边界；
6. 没有容器的文字最多可以得到多大的有限支持区域；
7. 哪些区域或拓扑无法可靠判断。

不需要识别正确 OCR 字符串，不需要生成文字像素 Mask、safe edit region 或清字范围。

## 2. 盲标纪律

标注前和标注过程中：

- 不看另一位标注者的答案；
- 不看 Detection、OCR、Grouping、Cleaning 或 association 输出；
- 不看旧 difference mask、旧 crop disposition 或 expected decision；
- 不因为你猜到 case 来源而查阅冻结答案；
- 不把“看起来可能可以清字”当作容器证据；
- 证据不足时写 `UNCERTAIN`，不得强制合并、分割、闭合或判成文字。

## 3. 保存叠加图

可以使用系统画图、Krita、GIMP 或其他不会自动改图的绘图工具。

对每个 case：

1. 打开 `images/case-XX.png`；
2. 立即“另存为”到自己的 `overlays/case-XX-overlay.png`；
3. 保持原始宽高，不裁剪、不旋转、不缩放画布；
4. 可以在软件里放大查看，但保存时图像尺寸必须不变；
5. 按下列颜色在副本上画中心线或轮廓。

## 4. 颜色和符号

| 标记 | 颜色 | 建议线宽 | 画法 |
| --- | --- | --- | --- |
| 文字组 `T1, T2...` | 橙色 `#FF6D00` | 2 px | 用矩形松散框住完整文字组，并在旁边写 `T1` 等编号。不要描字符像素。 |
| 容器可见边界 | 绿色 `#00C853` | 3 px | 沿实际可见边界画中心线。不要画成宽 Mask。 |
| 必要的虚拟边界 | 品红 `#D500F9` | 3 px | 只在缺口、遮挡、页面截断或容器接触分界处画最短合理中心线。 |
| 无容器文字支持区域 | 蓝色 `#2962FF` | 3 px | 画一个有限闭合 envelope；不得借机伪造气泡。 |
| 人工不确定区域 | 黄色 `#FFD600` | 3 px | 用闭合轮廓圈出；不要求半透明填充，并在表单解释原因。 |
| 页面边缘出口 | 白底红色 `#D50000` | 3 px | 在边界离开 ROI 的位置画短横杠；不要猜 ROI 外的路径。 |

颜色必须优先于线型。若工具无法精确设置色值，使用最接近且彼此可区分的纯色，并在表单说明。

## 5. 文字组规则

- 一个文字组是视觉上应共同阅读、方向一致或空间连续的一组字符/文字 fragment。
- 多列文字可以是多个 `T`，之后再判断它们是否属于同一个容器。
- 不要求抄写 OCR 内容，可用“左侧竖排”“中央时间数字”等位置描述。
- 如果疑似图案、网点或结构而非文字，选择 `NOT_TEXT`；不要为了完成表格创建 `T`。
- 文字被 ROI 裁断时仍可编号，但完整性填 `NO` 或 `UNCERTAIN`。

## 6. 容器与支持区域规则

每个 `T` 必须在表单中归入以下之一：

- `C1, C2...`：属于某个容器；
- `SUPPORT`：确有文字，但没有可支持的气泡/旁白框容器；
- `UNASSIGNED`：无法判断；
- `NOT_TEXT`：最终认为不是文字。

容器类型：

- `EXPLICIT`：可见轮廓足以支持真实容器；
- `IMPLICIT`：部分边界缺损、遮挡或被页面截断，但仍有足够局部证据；
- `UNCERTAIN`：是否存在容器或其拓扑无法可靠判断。

同一容器内不同文字组之间不要画虚拟边界。不同容器接触但分界证据不足时，不要强行找一条漂亮分割线；圈出黄色不确定区域并在表单选择 `GT_UNCERTAIN`。

## 7. 虚拟边界规则

- 只补证据支持的缺口，不沿你想象的完整气泡重画一圈。
- 有两个以上同样合理的闭合路径时，不任选其一：圈出不确定区域。
- 边界进入 ROI 外时，只标出口，不延伸到画布外。
- 虚拟边界是容器拓扑 GT，不是算法竞争 ridge，也不是清字区域。

## 8. 不确定与弃权

以下任一情况应选择 `GT_UNCERTAIN`：

- 无法确定是一个容器还是多个容器；
- 无法确定某个候选是不是文字；
- 同一位置存在多个同样合理的虚拟边界；
- 文字组归属只能靠语义猜测；
- 支持区域最大范围没有清楚的视觉依据；
- ROI 裁断导致关键证据不可见。

可以只把局部区域标为不确定，不必让整张 case 作废。

## 9. 每个 case 的完成条件

- 原始尺寸的 `case-XX-overlay.png` 已保存，或表单明确写明因 `NOT_TEXT` 无需叠加；
- 所有你认为是真实文字的区域都有 `T` 编号；
- 每个 `T` 都有容器/支持/未分配结论；
- 可见边界、必要虚拟边界和不确定区域已标出；
- 表单中的最终状态为 `CONFIDENT_GT` 或 `GT_UNCERTAIN`；
- 没有查看另一位标注者或 coordinator 材料。

边界 uncertainty band 的像素宽度由后续 harness 根据字符高度计算。你只画中心线和人工不确定区域，不需要计算 `delta`。
