# Pixel Evidence Review

1. **现有 `dilate` 风险**：`text_core_from_bbox` 以 bbox 内暗像素为种子，`expand_visible_text_support(..., 2)` 再作方形膨胀并直接充当 required；它能忠实暴露 g002/g004 与 physical boundary corridor 的冲突，却不能证明膨胀进入的像素是文字抗锯齿而非真实气泡轮廓，故不得缩小 required、扩大 safe，或以低 unsafe ratio 放行。
2. **A1 输入/输出**：输入 hash-lock 的 source、旧 required/text core、instance、protected、uncertainty、bbox 与局部背景统计；从高置信 core 作受限连通扩张，输出对每个旧 required 像素的 `required_text`、`proven_non_text_boundary` 或 `unresolved_uncertain` 分类、路径/局部外观证据及新候选支持（旧 required 保留为审计输入）。
3. **A1 guards**：扩张只能在 instance 内、从 core 连通、满足局部 Lab/亮度对比和有限 stroke/方向连续性的像素上进行；protected、已确认 boundary ridge、uncertainty 及其 guard band 是不可穿越硬障碍；任何分类为 `required_text` 的像素必须具有不跨越障碍的 seed path，且算法、阈值、配置完全同用于 g002、g004 与控制样本。
4. **A1 abstain**：候选路径触及 protected/ridge、局部背景样本不足、多个 core 竞争、连通性/外观证据矛盾，或任何旧 required 像素无法在三类中以证据解释时，标为 `unresolved_uncertain` 并 `INCOMPLETE_REVIEW`；它不得运行 Cleaner，也不得把“未扩张到”解释为边界。
5. **A2 输入/输出**：输入与 A1 相同，另从 source 计算多尺度 gradient ridge、方向连续性、曲率/闭合轮廓和内外光度跃迁；以高置信物理 boundary ridge 作为 geodesic barrier，输出 ridge-confidence/方向 artifact、旧 required 的三类映射与受 barrier 约束的 text-support candidate，而不修改原 protected mask。
6. **A2 guards**：只有 gradient、线连续性和 photometric transition 一致并与冻结 protected corridor 一致的 ridge 才可成为 hard barrier；任何 core 或候选文本与 ridge 重叠时不得自动判为 boundary，必须转 `unresolved_uncertain`；禁止把 physical boundary 改为 safe、借用 Slice F neighbor/virtual-boundary 语义、按 case/target/坐标分支，且 protected write、outside-safe write、boundary damage 任一大于零即候选 `NO_GO`。
7. **A2 abstain**：ridge 弱/断裂、交叉文字与轮廓不可分、封闭性或方向证据不足、小气泡分辨率不足、A1/A2 分类冲突，或人工审查未能保证 `false boundary-to-text = 0` 时，保留 blocker、Stage A `NO_GO`，不进入 Stage B；A2 的可行性仅是有界待证假设，并非已证明的 physical-boundary planner。
8. **测试**：先作只读 Stage A：复现 A0 的 g002=710、g004=70，逐组件 hash-lock 叠图与空白 `FORM-stage-a.md`；同配置比较 A1/A2/A3/A4，审计无 case-specific branch，并要求人工确认后的 false boundary-to-text=0。加入正控（case-71 interior、既有 virtual-boundary PASS、远离边界实例）无写域/残字/边界回归，负控（真触边文字、被 core 命中的边框、高梯度装饰框、小气泡、mixed component）全部 abstain。仅在 `unresolved_uncertain=0` 且全部 required_text 位于 safe 后，方可用独立 validator 从 bytes 重算 changed、protected/uncertainty/outside-safe/boundary damage、residue、halo、背景与 seam；任何硬失败均 `NO_GO`。
