# YOLO 开放词汇模型选择实验 — 最终结论

## 裁决

```text
VERDICT: NOT_ADOPTED
```

YOLO 开放词汇模型选择实验到此结束；它不进入本项目当前的 Detection/OCR 主线，也不进入 `DetectorProvider`、Workflow 或后续 Cleaning 方案。

## 已确认事实

- YOLOE-26N 与 YOLOE-11S 在固定的 12 页本地校准集上完成了 120/120 个 GPU/FP16/640 运行，运行本身没有失败；其中 68 个非空、52 个空结果。
- 120 行人工 overlay 审核均未填写，因此不存在可用于选择提示词、比较召回/误检或宣称检测质量的人工质量证据。
- YOLO-World V2.1 缺少官方 checkpoint 与可运行配置的精确映射；候选兼容性验证未进入可证明的 strict-load/GPU 推理阶段。
- 所有原始图像、权重、manifest、环境快照、运行输出、mask、overlay、CSV 和兼容性诊断仅保留在本地 `data/local/yolo-model-selection/`，不纳入 Git。

## 为什么不采用

本裁决不是“YOLOE 已被证明在所有漫画上无效”的模型质量结论。它是项目优先级与证据充分性的结论：该路线尚未形成可比较的人工质量证据，YOLO-World 对照也未就绪，而项目当前更需要推进端到端用户价值与已有 Detection/OCR 主线。

因此不得据此实验选择模型、调整正式检测架构、接入 Provider，或继续投入新的 YOLO 校准、模型矩阵与运行时兼容工作。未来若重新开启，必须以新的独立 Spike、重新定义目标、数据门禁和预算；不得把本地旧结果当作正式模型选择依据。
