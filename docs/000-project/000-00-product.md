# 产品定义

## 用户与价值

目标用户是希望阅读自己合法持有的日文漫画图片、但不参与专业汉化制作的普通读者。产品在本地把图片组织为 Project/Page，完成文字处理、中文翻译、基础清字、嵌字、检查与导出，并允许用户修改 OCR 或译文后局部返工。

## 首个产品 MVP

M1 Single-Page Visual Closure 是唯一的首个产品 MVP。它只要求在声明支持的普通对白气泡和旁白框范围内，对单张真实漫画页形成可操作、可复核、可导出的中文视觉闭环：可信 OCR/译文输入、容器关联、Cleaning/Check、Typesetting/Check、最小 Web 预览、人工修改、局部返工和单图导出。

产品级验收必须同时满足：原图不被覆盖；支持范围内的普通正文不静默缺失；失败可解释并保留原内容；修改后的依赖结果能够失效和局部重算；导出只使用当前已接受结果。局部 Spike 成功、FakeProvider 闭环或单个样本通过均不等于产品验收。

## 非目标

- 漫画搜索、抓取、下载、分发、发布或绕过第三方服务政策。
- 专业汉化组级精修、复杂拟声词、花体艺术字、复杂背景文字和任意排版复刻。
- 在 M1 中同时解决批处理、整章性能、所有 OCR/翻译质量和复杂恢复策略。
- 将开发期人工标注变成产品运行时必经步骤。

## 决策与理由

先验证单页视觉价值，再推进语义质量和规模化：这样能把失败定位在输入、关联、清字、嵌字或检查的具体边界，并避免把低质量结果放大到整章。拒绝“先做完整 Web/Batch 再补视觉质量”和“一次性要求全链路全自动达标”，因为两者都会扩大耦合与返工面。

## 风险、验证与开放问题

主要风险是支持范围被描述得比证据更宽、局部 warning 掩盖正文缺失，以及外部 Provider 拒绝被误当作可绕过错误。验证至少覆盖正常页、部分失败、Provider refusal、人工修改、stale propagation、局部返工和导出阻断。

开放问题：M1 的冻结验收样本集、BubbleInstance 的最小产品表达、最小 Web 入口的交互边界仍需在对应阶段设计中冻结。
